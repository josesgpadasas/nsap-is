History of the NSAP

The National Stock Assessment Program (NSAP) was first conceptualized in 1995 by Mr. Noel C. Barut, who was then the chief of the Pelagic Section of the Bureau of Fisheries and Aquatic Resources (BFAR) in the Philippines. It was institutionalized in 1997 as a regular program of BFAR to conduct assessments of major pelagic and demersal species in the country's fishing grounds. The program establishes standardized time series data on catch, effort, and biological parameters to support sustainable fisheries management. Over the years, NSAP has evolved to include interactive atlases, geographic systems, and partnerships for data collection, contributing to SDG 14.4 on regulating harvesting and ending overfishing.

What is the NSAP Information System?

The NSAP Information System is an upgraded, web-based version of the current NSAP Database, which was built using Microsoft Access. This new system leverages modern technologies to provide a secure, scalable, and user-friendly platform for managing fisheries stock assessment data. It allows real-time data entry, analysis, and reporting, replacing the limitations of desktop-based Access with cloud-hosted Supabase for backend storage and authentication. The system supports multiple user roles for controlled access, ensuring data integrity while enabling collaboration across regions.

Key Features

User Authentication & Roles

Secure login with roles: Superadmin (full control), Admin (manage users/data), Encoder (data entry), Viewer (read-only).

Data Management

Real-time entry and querying of stock assessment data, with regional filtering.

Reporting & Analytics

Generate reports, visualizations, and export data for analysis.

User Profiles

Personalized profiles with images, designations, and office details.

Cloud Storage

Upload and manage images/documents securely.

Responsive Design

Accessible on desktop, tablet, and mobile devices.
